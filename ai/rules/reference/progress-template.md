# PROGRESS.md

The format and procedure for the multi-session progress file is owned by the **`chisel-plan` skill** — see [ai/skills/chisel-plan/SKILL.md](../../skills/chisel-plan/SKILL.md).

That skill covers:

- When to create the file (thresholds)
- Filename rule (`PROGRESS.md` for non-Figma work: theme rebuild, multi-CPT feature, large refactor, content migration)
- Format template (with mode-appropriate `## Source` block)
- Typical phase sets (greenfield feature, rebuild, refactor)
- Hard rules (one status per phase, insert-don't-renumber, absolute dates, etc.)
- Procedure for create / update / session start + end
- Anti-patterns

For Figma imports specifically, the same skill produces `FIGMA_IMPORT_PROGRESS.md` — see [figma-import-template.md](figma-import-template.md).
