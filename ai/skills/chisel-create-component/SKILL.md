# Create Twig Component

**Load [reference/twig-templating.md](../../rules/reference/twig-templating.md) first** — Timber context, include syntax, naming conventions, where templates live (`views/`, not `custom/views/`). This skill is the _how_; reference is the _what_.

For Gutenberg editor content, use [create-pattern](../chisel-create-pattern/SKILL.md), [create-block](../chisel-create-block/SKILL.md), or [create-acf-block](../chisel-create-acf-block/SKILL.md) instead — components are for shared UI in PHP templates.

## Procedure

1. Create Twig template at `views/components/{name}.twig`
2. Create SCSS at `src/styles/components/_{name}.scss` — the build (`chisel-scripts`) auto-regenerates `src/styles/components/_index.scss` on next compile. **Do not edit `_index.scss` by hand** (header reads "auto generated — do not edit directly").
3. If interactive: create JS module at `src/scripts/modules/{name}.js`, import in `src/scripts/app.js`

## Twig template

```twig
{# Component: {Component Name} #}
{# @param {type} {param_name} - {description} #}

<div class="c-{name}">
  {% if param_name is defined and param_name %}
    <div class="c-{name}__element">{{ param_name }}</div>
  {% endif %}
</div>
```

Usage:

```twig
{% include 'components/{name}.twig' with { param_name: value } %}
```

## SCSS

```scss
@use '~design' as *;

.c-{name} {
  // BEM
  &__element { /* ... */ }
  &--modifier { /* ... */ }
}
```

## JS module (optional)

```js
class ComponentName {
  constructor(element) {
    this.element = element;
    this.initSelectors();
    this.initElements();
  }
  initSelectors() {
    this.selectors = {
      /* js-* */
    };
  }
  initElements() {
    this.elements = {
      /* cached */
    };
  }
}

export default ComponentName;
```

Import in `src/scripts/app.js`:

```js
import ComponentName from './modules/{name}';
new ComponentName();
```

## Conventions

- Twig: `{name}.twig`
- SCSS: `_{name}.scss` with `c-` prefix (or `o-` for atomic objects in `views/objects/`)
- JS selectors: `js-{name}`
- BEM: `.c-{name}`, `.c-{name}__element`, `.c-{name}--modifier`

## Objects (simpler, more abstract)

For atomic/abstract components (icon wrapper, badge primitive):

- Twig: `views/objects/{name}.twig`
- SCSS: `src/styles/objects/_{name}.scss` with `o-` prefix — `_index.scss` is auto-generated by the build, do not edit by hand
