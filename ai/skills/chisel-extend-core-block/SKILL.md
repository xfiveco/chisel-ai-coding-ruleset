# Extend Core Block

**Load [reference/blocks.md](../../rules/reference/blocks.md) first** — existing block styles, existing block mods, and where they live in `src/scripts/editor/`. This skill is the _how_; reference is the _what_.

Three approaches, simplest first:

1. **Block style** — CSS-only variant users can pick in editor sidebar
2. **Block mod** — toggle/select/input that adds a class or modifies rendering
3. **SCSS only** — always-on default styling without UI changes

## Decision

| Need                                | Approach                        | Effort |
| ----------------------------------- | ------------------------------- | ------ |
| Visual variant users pick           | Block style                     | Low    |
| On/off toggle that adds a class     | Block mod                       | Medium |
| New sidebar control (select, input) | Block mod                       | Medium |
| Always-on default styling           | SCSS only                       | Lowest |
| Complex rendering changes           | [create-block](../chisel-create-block/SKILL.md) | High   |

**Rule:** if the change should be the global default (not a variant), update the base style directly — see [adapt-base-styles](../chisel-adapt-base-styles/SKILL.md). Don't create a block style for something that should be the default.

## 1. Block Style

Adds a selectable style variant. User clicks → CSS class applied.

**Use for:** button colors, spacer sizes, heading decorations, card variants, list styles.

### Procedure

1. Edit `src/scripts/editor/blocks-styles.js` — add style to the relevant register method
2. Edit or create `src/styles/blocks/_core-{block-name}.scss` — style the `.is-style-{name}` class. `src/styles/blocks/_index.scss` is auto-generated by the build; new partials are picked up automatically — **do not edit `_index.scss` by hand**.

### JS pattern

```js
register{BlockName}Styles() {
  const styles = [
    { name: '{style-name}', label: __('{Label}', 'chisel'), isDefault: false },
    { name: '{another}', label: __('{Label}', 'chisel') },
  ];
  this.registerStyles('core/{block-name}', styles);
}
```

Then call `this.register{BlockName}Styles()` from `registerBlockStyles()`.

To remove a WordPress default first:

```js
unregisterBlockStyles() {
  setTimeout(() => {
    unregisterBlockStyle('core/{block-name}', ['default-style-to-remove']);
  }, 10);
}
```

### SCSS pattern

```scss
@use '~design' as *;

.wp-block-{block-name}.is-style-{style-name} {
  // variant styles
}
```

Class is always `is-style-{name}` where `{name}` matches the `name` property.

## 2. Block Mod

Adds custom toggle/select/input to block sidebar. Applies a class or modifies rendering.

**Use for:** disable bottom margin, toggle animations, visibility controls, layout options.

### Procedure

1. Create `src/scripts/editor/mods/{block-name}-{feature}.js` — see [templates/block-mod-template.md](../../rules/templates/block-mod-template.md) for full scaffold
2. Import in `src/scripts/editor/blocks-mods.js`: `import './mods/{block-name}-{feature}';`
3. If attribute needs REST/MCP validation, register server-side via filter in a `custom/app/WP/{Feature}.php` class (e.g. extend `custom/app/WP/Blocks.php` or create a new feature class) using the `HooksSingleton` trait — **never put hooks in `custom/functions.php`** (see [CLAUDE.md "Architecture"](../../../CLAUDE.md#architecture-core-vs-custom)):

```php
// custom/app/WP/Blocks.php (or similar feature class)
public function filter_hooks(): void {
    add_filter( 'register_block_type_args', array( $this, 'add_block_attribute' ), 10, 2 );
}

public function add_block_attribute( array $args, string $block_name ): array {
    if ( $block_name !== 'core/{block-name}' ) {
        return $args;
    }
    if ( ! isset( $args['attributes'] ) ) {
        $args['attributes'] = array();
    }
    $args['attributes']['{attributeName}'] = array(
        'type'    => 'boolean',
        'default' => false,
    );
    return $args;
}
```

Then bootstrap the class in `custom/functions.php` with `\Chisel\WP\Custom\Blocks::get_instance();`.

## 3. SCSS-Only Customization

Different default styling, no editor UI.

### Procedure

1. Edit or create `src/styles/blocks/_core-{block-name}.scss`. The barrel `src/styles/blocks/_index.scss` is auto-generated by the build and forwards new partials automatically — **do not edit `_index.scss` by hand**.

```scss
@use '~design' as *;

.wp-block-{block-name} {
  // base customizations
}
```
